왜 etlx인가

다른 ETL과 무엇이 다른가

Fivetran 같은 SaaS, dbt 같은 변환 전용 도구, Airflow 같은 오케스트레이터 — 각자 한 조각을 잘합니다. etlx는 그 경계를 없앱니다.

01

종속이 없습니다 (Agnostic)

특정 DB에도, 특정 오케스트레이터에도 묶이지 않는 순수 Python 코어. 같은 파이프라인을 Airflow·Dagster·Prefect 위에서, 또는 단독 CLI로 그대로 실행합니다.

대비 — 대부분의 SaaS ETL은 자사 플랫폼에, dbt는 SQL 변환에 묶입니다.
02

서비스는 선택입니다 (Service-optional)

코어 라이브러리는 웹 서비스를 전혀 모릅니다 — 단방향 의존. 웹 UI가 통째로 사라져도 라이브러리는 그대로 동작합니다. 필요한 만큼만 골라 쓰세요.

대비 — 웹 콘솔과 엔진이 한 덩어리라, 콘솔 없이는 못 쓰는 도구가 많습니다.
03

가장 싼 길을 알아서 고릅니다 (무이동 ELT)

소스와 싱크가 같은 DB면 데이터를 옮기지 않고 DB 안에서 한 문장으로 처리(푸시다운). 그게 안 되면 컬럼 형식(Arrow) 벌크 전송, 그것도 아니면 행 단위 — 작업마다 자동 선택하고 실제로 어느 길로 갔는지 보여줍니다.

대비 — 늘 행 단위로 옮기거나, 항상 한 가지 방식만 강제하는 엔진과 다릅니다.
04

리니지가 공짜입니다 (Zero-config)

실행만 하면 테이블·컬럼 단위 데이터 계보가 자동으로 그려집니다. SQL은 JOIN·CTE·집계·윈도우까지 전체 구문을 분석해 컬럼이 어느 원본에서 왔는지 추적하고, OpenLineage로 내보낼 수 있습니다.

대비 — 별도 카탈로그 도구를 붙이거나 수동으로 매핑해야 하는 경우가 흔합니다.
05

당신의 데이터는 당신의 인프라에 (오픈소스·셀프호스팅)

Apache 2.0 오픈소스. Docker Compose 한 줄 또는 Helm으로 당신의 서버·클러스터에 올립니다. 비밀정보는 Vault·AWS/GCP Secret Manager에 위임하고 DB엔 참조만 남깁니다.

대비 — SaaS ETL은 당신의 데이터가 외부 클라우드를 거칩니다.

한눈에

필요한 건 다 들어 있습니다

수집부터 변환, 적재, 자동화, 모니터링, 계보까지 — 하나의 플랫폼에서.

🔌

커넥터 20여 종

RDBMS·DW·NoSQL·스트리밍·오브젝트 스토리지를 동일한 컨트랙트로. 새 커넥터도 쉽게 확장.

⚡

자동 데이터 경로

푸시다운(무이동) · Arrow 벌크 · 행 단위를 작업마다 자동 선택. PG→PG 실측 약 3.4배.

🧬

컬럼 리니지

설정 없이 테이블·컬럼 계보 자동 생성. OpenLineage 내보내기로 표준 카탈로그 연동.

🧩

코드 & 노코드

YAML·Python으로, 또는 브라우저 비주얼 빌더로. 같은 엔진, 같은 결과.

🛡️

멱등성 & DLQ

두 번 돌려도 안전한 적재(덮어쓰기·병합). 실패한 행만 격리하고 전체는 살립니다.

⏱️

스케줄 · 센서

cron 정시 실행, 상류 도착·신선도 기반 자동 트리거. 자산 기반 오케스트레이션.

📈

수평 확장

PostgreSQL 큐(SKIP LOCKED) 멀티-워커. 대용량은 범위 분할로 병렬 백필.

🔭

모니터링

실행·로그·실패 분류·데이터 경로 가시화 + Grafana 대시보드 템플릿.

🚀

어디에나 배포

Docker Compose · Helm(쿠버네티스). 워커·스케줄러·센서 모두 멀티-replica 안전.

두 가지 모습

팀의 모두를 위해

한 엔진, 두 가지 사용법. 비개발자도 개발자도 같은 파이프라인을 다룹니다.

웹 UI

운영자 · 분석가

  • 브라우저에서 연결 등록 + 즉시 연결 테스트
  • 드래그 앤 드롭 비주얼 파이프라인 빌더
  • 버튼 한 번으로 실행, 실시간 진행·로그
  • 데이터 계보 카탈로그 · ERD 디자이너

라이브러리 · CLI

데이터 엔지니어

  • YAML 또는 Python으로 파이프라인 정의
  • etlx CLI · Airflow / Dagster / Prefect 어댑터
  • 커넥터·변환을 직접 확장 (Registry 패턴)
  • 타입 검증(Pydantic) · 단위/통합 테스트 1,300+
# orders_to_dw.yaml — 소스 → 변환 → 싱크
name: orders_to_dw
source: { connection: pg_prod, query: "SELECT * FROM orders" }
transforms:
  - { type: rename, mapping: { user_id: customer_id } }
  - { type: sql, query: "SELECT * FROM input WHERE total > 0", pushdown: true }
sink: { connection: warehouse, table: analytics.orders,
        mode: upsert, key_columns: [order_id] }   # 멱등

같은 정의를 CLI로 실행하거나, 웹 빌더로 그리거나 — 결과는 동일합니다.

시작하기

몇 분이면 충분합니다

저장소를 받고 동기화하면 끝. 필요한 커넥터만 골라 설치하세요.

# 1) 받기
git clone https://github.com/genebir/ETL && cd ETL
uv sync

# 2) 필요한 커넥터만
pip install 'etl-plugins[postgres]'
pip install 'etl-plugins[duckdb]'   # SQL 변환 + Arrow

# 3) 실행
uv run etlx run configs/pipelines/my.yaml --connections configs/connections.yaml
